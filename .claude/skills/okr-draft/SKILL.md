# OKR Draft Skill

四半期ごとに発生するOKR記入作業を効率化する。全社OKRシート(xlsx)を入力に、対象人物の **過去Quarterの記録**・**親OKR(FY/Q/ユニット)**・**同僚の記入例** を抽出し、それらと本人の意向を踏まえてObjectives + Key Resultsの記入案を生成する。

## いつ使うか

ユーザーが以下のような発話をしたとき:
- 「OKRの記入案を作って」「OKR書きたい」「OKRシート埋める」
- 「2026_Q3のOKRを書きたい」「次QのOKRどうしよう」
- 「四半期目標」「Objectives」「Key Results」「O&KR」のいずれかが文脈に出る
- OKRシート(xlsx)を渡してきた

注意: 単なる「OKRって何？」のような知識質問ではトリガーしない（記入案作成タスクの場合のみ）。

## 入力（対話で確定）

1. **xlsxファイルパス**
   - デフォルト: `C:\Users\YUKIHISA\Downloads\OKRシート_旧名_目標＆ウェイト整理.xlsx`
   - 命名は変動するため、Downloadsにある最新の `OKR*.xlsx` を確認するのが確実
2. **対象シート名**
   - 例: `2026_Q2`, `2026_Q3`, `2027_Q1`
   - 不明なら `extract_okr_context.py --list-sheets` でシート一覧を提示
3. **対象人物名**
   - 引数 or 対話で確認する（デフォルト固定はしない）
   - ユーザーが「自分の」「私の」と言った場合は、`memory/user_role.md` 等から本人の氏名を参照
   - 部分一致（姓または名のみ）でも引けるが、複数候補がある場合は確認

## 処理フロー

### 1. コンテキスト抽出

`scripts/extract_okr_context.py` を呼び出す:

```bash
python scripts/extract_okr_context.py \
  --file "<xlsxパス>" \
  --sheet "<シート名>" \
  --person "<対象人物名>"
```

スクリプトはJSONを stdout に出力し、以下を含む:
- `target_row_range`: 対象人物のC列結合範囲（例: [13, 20]）
- `target_existing`: 当該行の既存記入値
- `past_quarters[]`: 過去2 Quarter分の同人物のKR/着地/MORE
- `parent_okrs`: FY/Q/ユニットOKRのObjective + KR
- `peer_samples[]`: 同ユニット内の他メンバー記入例（書式・粒度の参考）

### 2. 文脈分析

JSONを読み、以下を整理する:
- **過去Quarterの文体パターン**: 例えば「`カテゴリ:具体内容`」形式の有無、Objective欄の使い方（空欄かフルテキストか）
- **積み残し**: 過去Quarterの「最終MORE」「最終着地=☁」項目から、当Quarterに引き継ぐべきテーマを抽出
- **親OKRからの逆算**: 当Quarter期間中に達成すべき親KRを特定し、対象人物の役割範囲で寄与できる項目を列挙
- **同僚との連携ポイント**: peer_samplesで対象人物の名前が言及されている箇所（例: 「○○さんとの連携」）

### 3. 方針ヒアリング

`AskUserQuestion` で本人に確認する。質問は4個まで、各選択肢は4個まで（ツール制約）:

1. **当Quarter固有の予定/状況**: 出張、リリース、検証イベント、組織変更など、過去から推測できない要素
2. **重点テーマ選択**: 過去持ち越し+親OKRから3-4個の候補を提示し、複数選択可
3. **Objective欄(D列)の扱い**: 空欄でKRのみ / Objectiveも記述
4. **出力形式**: Markdown確認 → 調整後TSV / 直接TSV / Markdownのみ

### 4. ドラフト生成

ヒアリング結果を踏まえて Objectives + KRs を生成。原則:
- **KR枠は8個まで**（OKRシートの設計上）。空枠を残してもよい
- **ウェイト合計は厳密に1.00**（ε < 1e-9）。検算必須
- **過去Quarterの文体に合わせる**（例: 「開発／〜: ...」 形式）
- **親KRと連動**: 各KRが親OKRのどのKRに寄与するかを意識
- **重点テーマほどウェイト大**
- **期日は当Quarter終了月末** が基本。前倒しが必要なものは個別設定

### 5. Markdown表で提示・レビュー

下記フォーマットでチャットに表示:

```markdown
## <対象人物> <シート名> OKR記入案

### Objective欄(D列)
（テキストまたは「空欄」）

### KR欄

| 行 | KR | 期日 | ウェイト | 備考 | 前提 |
|----|----|------|---------|------|------|
| R<開始> | ... | YYYY-MM-DD | 0.XX | ... | ... |
| ... |

ウェイト合計: 1.00
```

ユーザーの修正コメントを反映して再提示。OKが出たら次へ。

### 6. TSV書出

`scripts/write_okr_tsv.py` を呼び出し、`<Downloads>/<対象人物名>_okr_<シート名>.tsv` を生成（日本語ファイル名でOK。例: `唐子 征久_okr_2026_Q2.tsv` / `Yamada Taro_okr_2026_Q2.tsv`）:

```bash
python scripts/write_okr_tsv.py \
  --json - \
  --output "<出力パス>"
```

stdinで以下のJSONを渡す:

```json
{
  "krs": [
    {"kr": "...", "due": "YYYY-MM-DD", "weight": 0.10, "remark": "...", "premise": "...", "memo": ""},
    ...
  ]
}
```

スクリプトはBOM付きUTF-8 + CRLFでTSVを書出（Excel貼付互換）。

### 7. 貼り付け手順を案内

ユーザーに以下を示す:

1. **D列のObjective**: チャットのコードブロックを直接 D<開始行>セル に貼付（結合セル左上）。改行が崩れたら `Alt+Enter` で再現
2. **E-J列のKR**: TSVファイルをメモ帳で開いて全選択コピー → E<開始行>セルを選択して貼付（範囲展開される）

## 設計上の重要ポイント

### 結合セルの扱い
- OKRシートは1シートあたり 100〜150 個の結合セルがある
- 対象人物のC列・D列・K列・N列・Q列は縦結合（個人の行範囲全体に跨る）
- E〜J列は結合なし → TSV貼付OK
- D列は結合の左上セルに値が入るだけ（Excelが自動的に結合範囲全体を埋める）

### TSV出力の規則
- BOM付きUTF-8（ExcelがUTF-8を正しく認識）
- 改行コード CRLF
- セル内改行は使わない（KR・備考は1行テキスト）
- 区切り文字: タブ
- ダブルクォートは csv.QUOTE_MINIMAL（必要時のみエスケープ）

### Objective文の扱い
- TSVに含めない（改行を含むため貼付時に崩れやすい）
- 別途チャットのコードブロックで提示し、ユーザーがD<開始>セルに直接コピペ
- 結合セルの左上(D<開始>)に値を入れれば結合範囲全体に反映される

### 文体の尊重
- 過去Quarterの当人記入から文体パターンを抽出し、ドラフトに反映する
- 抽出する観点:
  - **KRの構造**: 「`カテゴリ:具体的な内容`」形式か、フリーテキストか
  - **使われる接頭辞**: `開発／〜`, `1.〜 2.〜` のような番号付け、絵文字の有無
  - **粒度**: 1KRあたりの文字数の目安（短文派 / 長文派）
  - **Objective欄(D列)を埋める習慣の有無**
- 同僚との連携文脈（例: `○○さんとの連携で〜`）も過去から学習

### 同僚名はさん付け
- 社内メンバーに言及する場合、必ず「○○さん」と表記する（呼び捨て禁止）
- 例: 「吉川との連携」 → 「吉川さんとの連携」
- 過去Quarterでも本人がそう書く慣習がある

### ウェイトの検算
- 必ず合計1.00（許容誤差 < 1e-9）
- 配分の感覚:
  - 最重要KR: 0.20-0.30
  - 中: 0.10-0.15
  - 副次的: 0.05

### Q1/Q4の最終MOREから持ち越しを抽出
- 「最終着地=☁（曇り）」かつ MOREに「実施できていない」「持ち越し」「未対応」とある項目は、本Quarterに継続するか方針判断する
- 機械的に全部入れず、本人の意向を確認したうえで取捨選択

## エラーハンドリング

- xlsxファイルが見つからない: Downloadsの最新 `OKR*.xlsx` を提示し選択させる
- シート名が存在しない: シート一覧を表示し選択させる
- 対象人物名がC列のどの結合セルにも該当しない: 部分一致で候補を提示
- 結合セルが期待と異なる: シート構造を再確認するよう警告

## 補助スクリプト

詳細は `scripts/` 配下のファイル冒頭コメント参照:

- `scripts/extract_okr_context.py` - xlsxから対象人物のコンテキスト抽出
- `scripts/write_okr_tsv.py` - JSONドラフトをTSVに書出

両スクリプトとも `openpyxl` に依存（`pip install openpyxl` 済みであれば使える）。
